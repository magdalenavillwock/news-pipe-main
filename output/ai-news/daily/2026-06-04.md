### Claude Code Releases

**[v2.1.162](https://github.com/anthropics/claude-code/releases/tag/v2.1.162)** (3. Juni 2026)

Ein außergewöhnlich umfangreiches Release mit neuen Features, UX-Verbesserungen und zahlreichen Bugfixes:

**Neue Features:**
- `claude agents --json` zeigt jetzt `waitingFor` — zeigt an, worauf eine wartende Session blockiert ist (z. B. Permission-Prompt)
- `--tools`: Explizites Auflisten von `Grep`/`Glob` aktiviert nun die dedizierten Such-Tools auf Native-Builds mit eingebetteter Suche (zuvor stillschweigend ignoriert)
- `/effort` bestätigt jetzt, wenn das gewählte Level als Standard für neue Sessions persistiert wird
- Klick auf Slash-Command im Autocomplete-Menü füllt ihn ins Prompt-Feld — Enter führt ihn aus (statt sofort auszuführen beim Klick)
- Remote Control erscheint als persistente Footer-Pill mit Link zur Session (statt als Startup-Meldung)
- Windsurf wurde in **Devin Desktop** umbenannt im `/ide`-Menü und verwandten Commands (Rebrand des Editors)

**Bugfixes (kritisch):**
- Stiller Startup-Hang behoben, wenn das Config-Verzeichnis schreibgeschützt ist — Claude Code startet jetzt mit In-Memory-Config und zeigt Fehler an statt leeren Bildschirm
- WebFetch Permission-Regeln wurden nicht auf vorgenehmigte Domains angewendet — `WebFetch(domain:...)` Deny/Ask/Allow-Regeln überschreiben jetzt korrekt die Auto-Allow-Liste
- Windows: Permission-Regeln matchten nie bei Backslash-Pfaden (`~\`, `\\server\share`) oder Groß-/Kleinschreibungsvarianten; Read-Deny-Regeln versteckten Dateien nicht aus Glob/Grep-Ergebnissen
- Interrupt (Esc) am Beginn eines Turns wurde in stream-json/SDK-Sessions stillschweigend verworfen — kein "Interrupted"-Feedback
- API-400-Fehler `no low surrogate in string` bei Emoji nahe Trunkierungsgrenzen behoben
- MCP per-Server `timeout` unter 1000 ms wurde auf 1-Sekunden-Watchdog gerundet und brach alle Tool-Calls ab — jetzt korrekt ignoriert (Fallback auf `MCP_TOOL_TIMEOUT`)
- LSP-Tool `workspaceSymbol` lieferte keine Ergebnisse — akzeptiert jetzt `query`-Parameter
- `claude agents` schnitt Status-Text (Tool-Args, Antworten, Prompts) bei 60–120 Spalten ab auf breiten Terminals — nutzt jetzt volle Terminal-Breite
- `claude agents` trunkierte lange Session-Namen bei 40 Spalten
- `claude agents` attach sprang nach Background-Service-Neustart sofort zurück zur Session-Liste
- Ctrl+V Bild-Paste in `claude agents` Dispatch-Input tat nichts
- Session-Hintergrundlegung per ← verlor Konversation bei nicht-startbarem Background-Service — bleibt jetzt als Failed-Row in der Liste
- Cross-Session Messaging (`SendMessage`) brach bei tiefen `CLAUDE_CODE_TMPDIR`/$TMPDIR-Verzeichnissen
- Öffnen einer laufenden Background-Session stagnierte 5 Sekunden
- `claude update` wartet jetzt auf Endpoint-Security-Scanning neuer Binaries (statt nach 5s zu scheitern)

**UX-Verbesserungen:**
- Ruhigerer Startup: Notices gruppiert nach Schweregrad, Session-Info und Announcements in einer Zeile
- Startup-Warnungen kürzer und mit konkretem Fix
- Launch-Prompt-Warnungen bleiben unterhalb des Inputs gepinnt
- Fehlgeschlagene Turns zeigen kompakte Warnzeile statt mehrzeiligem rotem Error-Block
- "Claude in Chrome enabled" und "marketplace installed" Startup-Meldungen entfernt

### Claude Agent SDK (TypeScript)

**[v0.3.162](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.3.162)** (3. Juni 2026)

Zwei wichtige Änderungen:

- **Strukturierte Refusal-Erkennung**: Refusal-Fehlermeldungen tragen jetzt `stop_reason: "refusal"` und `stop_details` auf der Assistant-Message und in Session-Transcripts. SDK-Konsumenten können Refusals damit programmatisch erkennen, ohne Textmuster zu matchen.
- **Standardmäßig eingebettete Suche**: Agent-SDK-Sessions auf Native-Builds nutzen jetzt standardmäßig das schnelle eingebettete `find`/`grep` in Bash (analog zur interaktiven CLI), statt immer dedizierte Grep/Glob-Tools zu registrieren. Wer die dedizierten Tools behalten möchte (z. B. für `canUseTool`-Hooks), listet sie in der `tools`-Option oder in `allowedTools` auf.

**Update:**
```sh
npm install @anthropic-ai/claude-agent-sdk@0.3.162
```

## 📄 ArXiv Papers

**[StreamMA: Streaming Communication in Multi-Agent Reasoning](http://arxiv.org/abs/2606.05158v1)**
Statt dem "generate-then-transfer"-Paradigma streamt StreamMA jeden Reasoning-Schritt sofort an nachgelagerte Agenten weiter. Das reduziert Latenz und verbessert überraschenderweise auch die Qualität — weil frühe Reasoning-Schritte zuverlässiger sind als spätere.

**[Reinforcement Learning from Rich Feedback with Distributional DAgger](http://arxiv.org/abs/2606.05152v1)**
Erweitert das RLVR-Rezept (bislang: einzelnes binäres Reward-Signal) um reichhaltigeres Feedback (Execution Traces, Tool Outputs, Expert Corrections). Nutzt eine Distributional-Variante von DAgger als effizientere Alternative zu RL mit Reward Models.

**[Failed Reasoning Traces Tell You What Is Fixable](http://arxiv.org/abs/2606.05145v1)**
Fehlgeschlagene Reasoning-Traces kodieren "Recoverability Structure" — ob ein Fehler durch mehr Sampling behebbar ist (unlucky) oder struktureller Natur ist (resistent gegen mehr Budget). Dieses Signal aus Fehltraces zu extrahieren könnte Test-Time-Compute gezielter einsetzen lassen.

**[STRIDE: Training Data Attribution via Sparse Recovery](http://arxiv.org/abs/2606.05165v1)**
Neue Methode zur Training-Data-Attribution für LLMs — verfolgt, welche Trainingsdaten zu welchen Modell-Vorhersagen beitragen, ohne wiederholtes Retraining per Sparse-Recovery-Ansatz.

**[Beyond Text Following: Repairable Arbitration Reversals in Audio-Language Models](http://arxiv.org/abs/2606.05161v1)**
Audio-Language-Models folgen oft konfligierendem Text statt klaren Audio-Signalen. In 64,1% der Konflikt-Samples ist die audio-gestützte Antwort repräsentiert aber unterdrückt — nicht grundsätzlich unverfügbar.
