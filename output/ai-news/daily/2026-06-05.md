### Anthropic & Claude Code Releases

#### [Claude Code v2.1.163](https://github.com/anthropics/claude-code/releases/tag/v2.1.163)

Das umfangreichste Release der letzten Tage mit zahlreichen Features und Fixes:

**Neue Features:**
- **`requiredMinimumVersion` / `requiredMaximumVersion`** — Enterprise-managed Settings: Claude Code verweigert den Start, wenn die eigene Version außerhalb des konfigurierten Bereichs liegt, und leitet Nutzer zur freigegebenen Version weiter.
- **`/plugin list` Kommando** — Listet alle installierten Plugins auf; unterstützt `--enabled`/`--disabled` Filter.
- **"c to copy" Shortcut in `/btw`** — Kopiert die rohe Markdown-Antwort in die Zwischenablage mit korrekter Formatierung.
- **Hooks-Erweiterung** — `Stop`- und `SubagentStop`-Hooks können jetzt `hookSpecificOutput.additionalContext` zurückgeben, um Claude Feedback zu geben und den Turn fortzusetzen, ohne als Hook-Fehler zu gelten.
- **Skills: `\$` Escape-Syntax** — Ermöglicht ein literales `$` vor einer Ziffer in Befehlskörpern.
- **stdio MCP Server** erhalten bei `--resume` dieselbe `CLAUDE_CODE_SESSION_ID` wie Hooks und Bash.

**Bugfixes:**
- `claude -p` hängte nach finalem Ergebnis nicht mehr, wenn ein Hintergrundprozess nie beendet wurde — Shells werden jetzt ~5s nach Ergebnis gestoppt.
- `claude -p` schlug auf Bedrock/Vertex/Foundry mit `CI=true` fehl ("ANTHROPIC_API_KEY required") — behoben.
- Bash-Kommandos schlugen unter Bazel und EDR-geschützten Go-Workflows fehl: `$TMPDIR` wurde fälschlicherweise global auf `/tmp/claude-{uid}` gesetzt (Regression in 2.1.154).
- Bash-Kommandos auf Windows schlugen mit "EEXIST" fehl, wenn das Session-Verzeichnis Read-Only oder in OneDrive war.
- Org-managed Permissions wurden nicht für die gesamte Session angewendet, wenn der Fetch während des Starts auf einem neuen Config-Verzeichnis abschloss.
- Hintergrund-Sessions in `claude agents` verloren laufende Tasks nach Claude Code Update und Reattach.
- Terminal-Fehlausrichtung und mehrsekundiger Hang beim Verlassen der Agent-View via Esc.
- Stop-Button bei Desktop-App-Chips räumte den Chip nicht auf, wenn der Prozess bereits beendet war.
- Tastatureingabe nach Paste-Operation mit fallengelassenem End-Marker dauerhaft unresponsiv.
- Hook `if: "Bash(...)"` Bedingungen feuerten bei jedem Bash-Befehl mit `$()` oder `$VAR`.
- Deny-Regeln für Home-Directory-Pfade (z.B. `Read(~/Desktop/**)`) blockierten keine Bash-Kommandos mit `$HOME`.
- Stray "(no content)" Zeile im Transcript nach Schließen von Panel-Dialogen.

**Verbesserungen:**
- Hintergrund-Agent-Sessions updaten Claude Code im Hintergrund — kein Cold Restart beim Öffnen.
- Klarere Beschreibungen für Built-in Commands und Skills im `/`-Menü.
- Subscription-Switch-Suggestion erscheint jetzt im Startup-Announcement-Slot statt als Toast.
- `claude agents` Dispatching startet Session im Verzeichnis, aus dem der Agent-View geöffnet wurde.

#### [Claude Code v2.1.165](https://github.com/anthropics/claude-code/releases/tag/v2.1.165)

- Bug fixes und Stabilitätsverbesserungen (kleines Patch-Release).

### 📄 ArXiv Papers

**[HANDOFF: Humanoid Whole-Body Control](http://arxiv.org/abs/2606.06493v1)** — Neues Framework für Humanoid-Roboter mit kompaktem Interface zwischen Task-Planung und Ganzkörper-Steuerung. Überbrückt die Lücke zwischen sprachlichen Task-Semantiken und kinematischen Referenzen für diverse Manipulationsfähigkeiten.

**[Code2LoRA](http://arxiv.org/abs/2606.06492v1)** — Hypernetwork-Framework, das repository-spezifische LoRA-Adapter generiert und so Repository-Wissen ohne Inference-Token-Overhead injiziert. Adressiert das Problem teurer Per-Repository-Fine-Tuning bei wachsenden Codebases.

**[TempoVLA](http://arxiv.org/abs/2606.06491v1)** — Geschwindigkeits-kontrollierbare Vision-Language-Action-Policies für Roboter-Manipulation: Schnell in risikoarmen Transit-Phasen, langsam und präzise bei Kontakt-kritischen Momenten — statt eines einzigen fixen Tempos wie bisher.

**[MLEvolve](http://arxiv.org/abs/2606.06473v1)** — Self-Evolving Multi-Agent Framework für automatisierte ML-Algorithmen-Entdeckung. Löst Probleme wie Inter-Branch-Isolation und gedächtnislose Suche in Long-Horizon-Optimierungen.

**[OpAI-Bench](http://arxiv.org/abs/2606.06481v1)** — Neuer Benchmark zur KI-Text-Detektion mit Fokus auf progressive Human-AI-Co-Editing-Workflows. Analysiert, wie KI-Authorship-Signale entstehen, akkumulieren oder verschwinden — nicht nur bei Endprodukten.

**[YOIO: Cross-Layer Sparse Attention](http://arxiv.org/abs/2606.06467v1)** — "You Only Index Once" — Shared Routing über Transformer-Schichten für effizientere Sparse Attention in Long-Context-Reasoning ohne die üblichen Qualitäts-Verluste strukturierter Block-Sparse-Methoden.
