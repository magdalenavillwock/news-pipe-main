### Anthropic & Claude

#### [Claude Code v2.1.179](https://github.com/anthropics/claude-code/releases/tag/v2.1.179)

Neun Fixes in diesem Release:

- **Mid-stream Connection Drops behoben** — Bei unterbrochenen Verbindungen werden Partial-Responses nun bewahrt statt als Rohfehler angezeigt. Der Spinner bleibt nicht mehr bei „running tool" stecken.
- **WSL2 Mouse-Wheel-Scrolling** — Regression aus v2.1.172 behoben; Scrollen in Windows Terminal und VS Code unter WSL2 funktioniert wieder.
- **Linux Sandbox Crash** — Ein `denyRead`/`allowRead`-Glob über einen großen Verzeichnisbaum machte die Bash-Tool-Beschreibung riesig und die Session unbrauchbar. Behoben.
- **Feedback-Survey-Bug** — Eine einstellige Antwort wurde fälschlicherweise als Session-Rating direkt nach einem Turn gewertet.
- **Welcome Screen** — Mehrere Promo-Banner konnten sich stapeln; jetzt erscheint höchstens ein Banner pro Session.
- **Ctrl+O für Subagenten** — Das Öffnen des Subagenten-Transkripts per Ctrl+O funktioniert jetzt korrekt.
- **Fokus-Verlust behoben** — Klick auf das Prompt-Eingabefeld gibt den Fokus wieder zurück (vom Subagent/Footer-Panel).
- **Remote Background Tasks** — Tasks erschienen zwischen Turns fälschlicherweise als „still running". Behoben.
- **Plugin Loading** — Performance beim Laden von Plugins in Remote-Sessions verbessert.

#### [Claude Agent SDK TypeScript v0.3.179](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.3.179)

- **Neues `tool_use_meta`-Sidecar** — Assistenten-Messages können nun ein optionales `tool_use_meta`-Objekt enthalten mit menschenlesbaren Display-Namen für Tool-Calls. SDK-Konsumenten können damit lesbare Labels rendern statt Wire-Namen.
- **`-p`-Mode Exit-Bug behoben** — Der Prozess beendete sich, bevor die Benachrichtigung eines abgeschlossenen Background-Agents zugestellt wurde, wodurch Zwischentext als Endergebnis ankam.
- **Remote-Session Busy-State** — Remote (stream-json) Sessions wurden für die gesamte Dauer eines Background-Workflows als „busy" angezeigt. Das Turn-Ergebnis wird jetzt an der Turn-Grenze emittiert; die Session meldet idle, während Background-Tasks weiterlaufen.

### Big Labs — AWS

#### [Amazon Bedrock InvokeGuardrailChecks API](https://aws.amazon.com/blogs/machine-learning/safeguard-your-agentic-ai-applications-with-the-amazon-bedrock-guardrails-invokeguardrailchecks-api/)

Neue API für granulare Sicherheitschecks in Agenten-Loops:

- **Ressourcenlos** — Keine GuardrailIDs oder Versionsverwaltung nötig; Checks werden direkt im Request spezifiziert.
- **Detect-only mit Scores** — Gibt numerische Schweregrad-Scores (0–1) zurück; die Anwendungslogik entscheidet über Block/Log/Eskalation.
- **Unterstützte Checks**: Content Filter (HATE, VIOLENCE, SEXUAL etc.), Prompt Attack Detection (Jailbreak, Injection, Leakage — jetzt als eigenständiger Check), Sensitive Information (31 PII-Typen).
- **Agentic-Fokus** — Ermöglicht unterschiedliche Sicherheitsprüfungen je nach Schritt im Agent-Loop (User-Input, Tool-Output, Final Response).

Integration mit Strands Agents via Lifecycle-Hooks demonstriert.

#### [SageMaker Container Image Caching](https://aws.amazon.com/blogs/machine-learning/introducing-container-caching-in-amazon-sagemaker-ai-for-faster-model-scaling/)

Container-Images werden lokal gecacht — eliminiert den Image-Pull-Schritt beim Launch neuer Instances:

- Scaling-Latenz für Qwen3-8B auf ml.g6.2xlarge: von 525s auf 258s (**-51%**)
- Frühe Kunden: bis zu **-65%** Verbesserung
- Kein Opt-in nötig: aktiviert sich automatisch auf unterstützten Accelerator-Instances

Ergänzt bestehende Optimierungen (Sub-Minute CloudWatch Metrics, Inference Component Data Cache).

#### [P-EAGLE auf SageMaker JumpStart](https://aws.amazon.com/blogs/machine-learning/parallelize-speculative-decoding-with-p-eagle-on-amazon-sagemaker-ai/)

AWS hat P-EAGLE (Parallel-EAGLE) entwickelt und open-source beigetragen — eine Methode, die Speculative Decoding von sequenziell auf vollständig parallelisiert umstellt:

- **Kernidee**: Statt K sequenzieller Drafting-Passes werden alle K Draft-Tokens in einem einzigen Forward Pass erzeugt (via learnable Placeholder-Embeddings).
- **Performance**: bis zu **1,69× Throughput** über EAGLE-3 auf NVIDIA B200
- **Tiefere Spekulation kostenlos**: K=3 und K=7 kosten denselben Pass.
- Verfügbar in JumpStart für: GPT-OSS-120B, GPT-OSS-20B, Qwen3-Coder-30B-A3B-Instruct, Gemma-4-31B-IT

### 📄 ArXiv Papers

**[VERITAS: Visual Verification für Roboter-Policies](http://arxiv.org/abs/2606.18247v1)**
Generator-Verifier-Framework für generalistische Roboter-Policies: Ein vortrainiertes Policy-Netz wird mit einem gradienten-freien visuellen Verifier kombiniert, der Aktionen zur Inference-Zeit bewertet und selbstverbessernd wirkt.

**[ReproRepo: Reproduzierbarkeits-Audits mit GitHub Issues](http://arxiv.org/abs/2606.18237v1)**
Framework, das GitHub-Issues als natürlich entstehende Supervision für Reproduzierbarkeits-Benchmarks nutzt — skalierbar ohne manuellen Kurations-Aufwand.

**[Looped World Models (LoopWM)](http://arxiv.org/abs/2606.18208v1)**
Erste Looped Architectures für World Modelling: Ein parameter-geteilter Transformer-Block verfeinert latente Umgebungszustände iterativ. Bis zu **100× Parametereffizienz** gegenüber konventionellen Ansätzen bei adaptiver Compute-Allokation.

**[Variable-Width Transformers](http://arxiv.org/abs/2606.18246v1)**
Empirische Untersuchung nicht-uniformer Kapazitätsverteilung über Netztiefe: Eine ×-förmige Architektur beginnt breit, wird in der Mitte schmal und endet wieder breit — verschiedene Schichten spielen unterschiedliche rechnerische Rollen.

**[Zone of Proximal Policy Optimization (ZPPO)](http://arxiv.org/abs/2606.18216v1)**
Verbindet Knowledge Distillation mit RL: Wenn alle Rollouts eines kleinen Students fehlschlagen (zero advantage), werden Teacher-Prompts als Scaffold injiziert — statt die Samples stumm zu verwerfen.

**[EvolveNav: Self-Evolving Memory für Zero-Shot Navigation](http://arxiv.org/abs/2606.18235v1)**
Embodied Agents bauen zur Test-Zeit eine agentic rule memory auf, indem sie Aktionswissen aus vergangenen Erfahrungen extrahieren — kontinuierliche Verbesserung ohne Re-Training.

**[DeLM Paper (Stanford)](http://arxiv.org/abs/2606.10662)** *(via VentureBeat referenziert)*
Dezentrales Multi-Agenten-Framework; Details siehe VentureBeat-Abschnitt.

**[Darshana Graph: Indische Philosophie-Corpus](http://arxiv.org/abs/2606.18222v1)**
125.000+ Text-Records aus klassischen hinduistischen, buddhistischen und jainistischen Texten — inkl. 8.500 Datensätzen mit demselben Ur-Vers kommentiert über 18 historische Kommentatoren.
