### Anthropic & Claude

#### [Claude Code v2.1.169](https://github.com/anthropics/claude-code/releases/tag/v2.1.169) *(high priority)*

**Neue Features:**
- **`--safe-mode` / `CLAUDE_CODE_SAFE_MODE`**: Startet Claude Code ohne alle Anpassungen (CLAUDE.md, Plugins, Skills, Hooks, MCP-Server) — ideal für Troubleshooting.
- **`/cd`-Befehl**: Wechselt das Arbeitsverzeichnis einer Session, ohne den Prompt-Cache zu unterbrechen.
- **`disableBundledSkills` / `CLAUDE_CODE_DISABLE_BUNDLED_SKILLS`**: Versteckt eingebettete Skills, Workflows und Built-in Slash-Commands vor dem Modell.
- **`/workflows` öffnet sofort**, auch während ein Turn läuft.
- **`TaskCreate`-Zuverlässigkeit verbessert**: Fehlerhafte Eingaben werden automatisch repariert; Validierungsfehler für ungeladene Tools enthalten nun das Schema.
- **CLAUDE.md-Warnschwelle** skaliert jetzt mit dem Kontextfenster des Modells.
- **Tipp für `claude agents`** wird angezeigt, wenn mehrere Sessions gleichzeitig laufen.
- **Background Sessions** übernehmen jetzt `--ide`, `--chrome`, `--bare`, `--remote-control` etc. über retire→wake hinaus.
- **5-Minuten-Idle-Timeout** auf Vertex/Foundry wiederhergestellt (opt-out: `API_FORCE_IDLE_TIMEOUT=0`).
- **Remote-managed Settings** mit ungültigem Eintrag wenden verbleibende gültige Policies an und zeigen den Fehler an.
- **Auto-Updater auf Windows** stoppt Wiederholungsversuche innerhalb einer Session, wenn `claude.exe` gesperrt ist.
- Verbesserte Farbkontraste für Skill-Tags im Slash-Command-Menü.

**Bugfixes:**
- Up/Down-Pfeile springen nicht mehr direkt in die Command History, wenn eine Eingabezeile umgebrochen ist — durchlaufen jetzt jede visuelle Zeile.
- Enterprise-MCP-Policies (`allowedMcpServers`/`deniedMcpServers`) werden jetzt auch bei Reconnect, IDE-Configs, `--mcp-config` und vor dem Laden von Remote-Settings erzwungen.
- ~30–50ms UI-Stall zu Beginn jedes Turns für macOS-Nutzer mit claude.ai-Credentials behoben.
- `claude -p` auf Windows war langsam/hing (Regression in 2.1.161) — gefixt.
- Remote Control blieb nach OAuth-Token-Refresh bei "reconnecting" hängen — gefixt.
- Git Credential Manager öffnete bei Windows-Start "Connect to GitHub"-Popup — gefixt.
- Footer-Hints (z.B. "esc to interrupt") wurden bei Custom Statusline nicht angezeigt.
- Veraltete Permission/Dialog-Prompts erschienen bei Wiederanbindung an Remote-Sessions erneut — gefixt.
- `claude agents --json` ließ blockierte und gerade dispatchtete Sessions weg; neues `--all`-Flag für abgeschlossene Sessions, neue `id`- und `state`-Felder.
- Agents-Ansicht hinterließ auf WSL in Windows Terminal einen veralteten/korrumpierten Frame.
- Background Agents ignorierten `env`-Werte aus Project-Level-Settings (z.B. `ANTHROPIC_MODEL`).
- MCPB-Plugin-Cache wurde auf Windows fälschlicherweise invalidiert.
- Plugin `.in_use` PID-Lock-Dateien häuften sich unbegrenzt an; veraltete Marker von abgestürzten Sessions werden jetzt einmal täglich bereinigt.
- Nicht vertrauenswürdige Projekt-Settings konnten OTEL-Clientzertifikatpfade ohne Bestätigung setzen.
- CPU-Auslastung bei streamenden Antworten und Spinner-Animationen reduziert.

#### AWS AI Blog — Mehrere Beiträge

**[AgentCore Runtime: Coding Agents hosten](https://aws.amazon.com/blogs/machine-learning/its-safe-to-close-your-laptop-now-hosting-coding-agents-on-amazon-bedrock-agentcore/)**
Amazon Bedrock AgentCore Runtime gibt jedem Agent-Session eine isolierte Firecracker-MicroVM mit persistentem `/mnt/workspace`. Features:
- Persistenter Workspace überlebt Stop/Resume (14 Tage Inaktivitätsspeicher)
- Interaktive Shell via `agentcore exec --it` mit PTY, Tab-Completion, Reconnect
- Deterministische Command-Ausführung ohne Modell im Loop (`InvokeAgentRuntimeCommand`)
- S3/EFS-Mounts für geteilte Skills und Caches
- **AgentCore Gateway**: Ein MCP-Endpunkt für GitHub, Jira, Slack etc. — Credentials nie im Agent
- **AgentCore Identity**: Bot-Pattern, On-behalf-of OAuth 2.0 (RFC 8693), Broker-Pattern
- Unterstützt Claude Code, Codex, Kiro, Cursor CLI, Gemini CLI in parallelen MicroVMs
- Kunden: Thomson Reuters, Iberdrola, Cox Automotive, Druva

**[EU Cross-Region Inference auf Amazon Bedrock](https://aws.amazon.com/blogs/machine-learning/unlocking-ai-flexibility-in-europe-a-guide-to-cross-region-inference-for-eu-data-processing-and-model-access/)**

**[Amazon QuickSight ARNs & Cross-Account Migration](https://aws.amazon.com/blogs/machine-learning/amazon-quick-arns-cross-account-migration-and-namespace-permissions/)**

### 📄 ArXiv Papers

**[OmniGameArena: VLM Game Agents Benchmark (UE5)](http://arxiv.org/abs/2606.09826v1)**
Einheitliches Benchmark für Vision-Language-Modell-Agenten in 12 neuen Unreal Engine 5-Spielen. Vergleicht kommerzielle VLMs, Open-Weight-VLMs und spezialisierte Game-Policies auf gleichem Footing, inklusive Multi-Agent- und Verbesserungsdynamiken.

**[Agency-Transferring Model-Free Policy Enhancement](http://arxiv.org/abs/2606.09825v1)**

**[iOSWorld: Benchmark für persönlich intelligente Phone Agents](http://arxiv.org/abs/2606.09764v1)**
Erster interaktiver nativer iOS-Simulator-Benchmark mit persistenter Nutzeridentität über 26 selbst entwickelte iOS-Apps. Agents müssen auf Transaktionen, Nachrichten und persönliche Präferenzen des fiktiven Nutzers schließen.

**[Collaborative Human-Agent Protocol (CHAP)](http://arxiv.org/abs/2606.09751v1)**
Protokoll für Multi-Human-Multi-Agent-Kollaborationen über Teams, Zeitzonen und Vertrauensgrenzen hinweg. Adressiert Produktions-Deployments, bei denen Agenten Verantwortung für Code, Verträge und klinische Entscheidungen tragen.

**[Evaluation Cards: Interpretive Layer für AI Evaluation](http://arxiv.org/abs/2606.09809v1)**
Standardisiertes Format für AI-Evaluationsberichte — überbrückt Inkonsistenz zwischen Leaderboards, Model Cards und Benchmark-Papieren. Ermöglicht zuverlässigere Vergleiche und Nachverfolgbarkeit.

**[PTL-Diffusion: Manifold-Aware Diffusion mit Periodic Terminal Laws](http://arxiv.org/abs/2606.09816v1)**
Diffusionsmodelle mit strukturierteren Terminal-Verteilungen für Daten auf niederdimensionalen Mannigfaltigkeiten — explizitere Struktur als Standard-Gauss-Referenzgesetze.

**[Learnability von Formal Language Tasks (Kausal)](http://arxiv.org/abs/2606.09822v1)**
Kontrolliertes Setting (probabilistische endliche Automaten) zur Untersuchung, wie viel aufgabenspezifische Daten zum Erlernen einer Aufgabe nötig sind — eliminiert Confounding durch natürliche Sprache.
