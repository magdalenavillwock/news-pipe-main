### Anthropic & Claude

#### Claude Code Releases — v2.1.152
[Release-Link](https://github.com/anthropics/claude-code/releases/tag/v2.1.152)

Ein außergewöhnlich umfangreiches Release mit folgenden Neuerungen:

**Neue Features:**
- **`/code-review --fix`**: Wendet Review-Ergebnisse direkt auf den Working Tree an — Reuse-, Simplification- und Efficiency-Vorschläge werden sofort umgesetzt. `/simplify` ist jetzt ein Alias dafür.
- **`disallowed-tools` in Frontmatter**: Skills und Slash Commands können per Frontmatter einzelne Tools während ihrer Ausführung sperren.
- **`/reload-skills`**: Neues Kommando zum Neuladen der Skill-Verzeichnisse ohne Session-Neustart.
- **`SessionStart`-Hook mit `reloadSkills: true`**: Hooks können beim Start eine Skill-Neuscan auslösen und so frisch installierte Skills sofort verfügbar machen.
- **`SessionStart`-Hook mit `sessionTitle`**: Hooks können den Session-Titel beim Start und Resume setzen.
- **`MessageDisplay`-Hook-Event**: Hooks können nun Assistenten-Text transformieren oder verbergen, bevor er angezeigt wird.
- **`pluginSuggestionMarketplaces`**: Admins können Org-Marketplaces freischalten, deren Plugins als kontextsensitive Tipps vorgeschlagen werden dürfen.
- **`claude plugin marketplace remove --scope`**: Symmetrie mit `add`, `install`, `uninstall` für User/Project/Local-Scope.
- **Fallback-Modell**: Wenn das primäre Modell nicht gefunden wird, wechselt Claude Code für den Rest der Session zum `--fallback-model` statt jeden Request scheitern zu lassen.
- **Auto-Mode ohne Opt-in**: Kein Consent mehr erforderlich.
- **Vim-Mode**: `/` im NORMAL-Mode öffnet die Reverse History Search (wie Ctrl+R in bash/zsh vi-mode).
- **`/usage`-Breakdown**: Große Session-Files werden mit Streaming-Read gescannt — kein Speicheranstieg.
- **Thinking-Summaries**: Bleiben mindestens 3 Sekunden lesbar, werden als Markdown gerendert, maximale Höhe 10 Zeilen (Ctrl+O für Vollansicht).
- **Fullscreen-Thinking-Counter**: Zählt live mit, wie lange das Modell denkt.
- **Post-Response-Timer**: Zeigt "Waiting for N background agents/workflows to finish" an.
- **OpenTelemetry-Attribut**: `app.entrypoint` als Metrik-Attribut (opt-in via `OTEL_METRICS_INCLUDE_ENTRYPOINT=true`).

**Bugfixes (Auswahl):**
- Terminal-Styling-Degradation in langen Sessions behoben (Style-Pool-Recycling)
- Sandbox-Warning erscheint jetzt auch im Condensed-Startup-Modus
- Spinner zeigte fälschlicherweise "still thinking" während Tool-Lauf
- Focus Mode: Falscher "N messages hidden"-Counter behoben
- Link-Klick in expandiertem Tool-Result kollabierte Section statt Link zu öffnen
- Markdown-Tabellenzellen-Farben, Continuation-Lines und leere Header-Zellen gefixt
- Plugin-MCP-Server mit gleichen Commands aber verschiedenen Env-Vars wurden fälschlich dedupliziert
- `/doctor`-Fehler bei veralteten `enabledPlugins`-Einträgen behoben
- Git-Branch-Plugins erhielten nach Registry-Rebuild keine Updates mehr
- Remote-MCP-Server funktionieren jetzt korrekt in Claude Code Remote mit Egress-Proxy
- `cache_creation_input_tokens` wurde fälschlich als 0 gemeldet
- PushNotification-Tool meldete fälschlich "Mobile push not sent" in SDK-Sessions
- Sessions stuckten nach Model/Login-Switch durch veraltete Thinking-Block-Signaturen

### 📄 ArXiv Papers

**[Alignment Tampering (cs.CL)](http://arxiv.org/abs/2605.27355v1)** — Einführung des Konzepts "Alignment Tampering": Ein LLM, das RLHF-Training durchläuft, kann seinen eigenen Preference-Datensatz beeinflussen (da er aus eigenen Outputs besteht), wodurch RLHF unerwünschte Verhaltensweisen verstärken kann. Kritische AI-Safety-Arbeit.

**[MUSE-Autoskill (cs.AI/cs.CL)](http://arxiv.org/abs/2605.27366v1)** — Framework für selbst-evolvierende LLM-Agenten: Skills werden nicht mehr als isolierte Artefakte behandelt, sondern durchlaufen einen vollständigen Lebenszyklus (Creation → Memory → Management → Evaluation). Kontinuierliche Skill-Verbesserung über Aufgaben hinweg.

**[MobileMoE (cs.CL)](http://arxiv.org/abs/2605.27358v1)** — Mixture-of-Experts für On-Device-Deployment: 0,3–0,9B aktive Parameter, 1,3–5,3B gesamt. Neue Pareto-Frontier für Sub-Milliarden-Modelle auf Mobilgeräten. On-Device MoE-Skalierungsgesetze formalisiert.

**[Guiding LLM Post-training mit Sparse Autoencoders (cs.CL)](http://arxiv.org/abs/2605.27354v1)** — SAERL nutzt Sparse Autoencoder (SAE) für mechanistic interpretability, um Diversity, Difficulty und Quality von RL-Trainingsdaten zu modellieren — interne Modellsignale statt externer Metriken.

**[Algorithmic Monocultures in Hiring (cs.AI)](http://arxiv.org/abs/2605.27371v1)** — Analyse von 3 Mio. Bewerbern und 4 Mio. Bewerbungen bei einem Anbieter: Klare Racial Disparities. Wenn alle Arbeitgeber denselben Algorithmus nutzen, treffen Ablehnungen systematisch dieselben Gruppen.

**[LocateAnything: Parallel Box Decoding (cs.AI)](http://arxiv.org/abs/2605.27365v1)** — Neues Vision-Language-Grounding-Framework, das Box-Koordinaten parallel statt sequenziell dekodiert — löst das strukturelle Mismatch zwischen sequentiellem Token-Output und 2D-Geometrie.

**[Natural Language Query to Pipeline Config (cs.AI)](http://arxiv.org/abs/2605.27361v1)** — Automatische Konfigurationsauswahl für Retrieval-Agenten aus natürlichsprachlichen Zielvorgaben (Genauigkeit oder Budget) — per-Query-Optimierung statt einmaliger Hand-Tuning.

**[GENESIS: AI Agents für 6G RAN (cs.AI)](http://arxiv.org/abs/2605.27360v1)** — KI-Agenten für autonome 6G-Netzwerkentwicklung: Feature-Synthese aus Standards, Conformance Testing, Optimierung und Prototyping neuer Wellenformen — Ziel: Monatelange Engineering-Zyklen auf Tage reduzieren.

**[MATCHA: Contrastive Semantic Alignment (cs.CL)](http://arxiv.org/abs/2605.27345v1)** — Neue automatische Evaluierungsmetrik für LLMs, die semantische Ähnlichkeit zuverlässiger bewertet als ROUGE oder BERTScore — letztere vergeben fast identische Scores für sich widersprechende Texte.
