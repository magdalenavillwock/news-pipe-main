#### Claude Code Releases — [v2.1.198](https://github.com/anthropics/claude-code/releases/tag/v2.1.198)

**Neue Features:**
- **Claude in Chrome ist Generally Available** — offizieller GA-Launch des Chrome-Integrations.
- **Background Agent Notifications** — Wenn ein Agent in `claude agents` auf Input wartet oder fertig ist, feuert jetzt der `Notification`-Hook mit den Events `agent_needs_input` / `agent_completed`.
- **`/dataviz` Skill** — Neuer Built-in-Skill für Chart- und Dashboard-Design-Guidance, inklusive ausführbarem Color-Palette-Validator.
- **Gateway: AWS als Provider** — Claude Platform on AWS (`anthropicAws`) wurde als Upstream-Provider hinzugefügt; model-not-found-Antworten rücken die Failover-Chain weiter.
- **Automatische Draft-PRs aus Background Agents** — Agents, die aus `claude agents` gestartet werden und Code-Arbeit in einem Worktree abschließen, committen, pushen und öffnen automatisch einen Draft-PR, statt zu stoppen und nachzufragen.
- **Explore Agent erbt Session-Modell** — Der eingebaute Explore-Agent übernimmt jetzt das Modell der Hauptsession (bis max. Opus) statt auf Haiku zu laufen.
- **Extended Thinking für Subagents** — Subagents und Context Compaction erben die Extended-Thinking-Konfiguration der Session → bessere Ausgabequalität bei delegierten Tasks.
- **Verbesserter Focus Mode** — Subagents, die in einem Turn gestartet werden, erscheinen in dessen Activity Summary; abgeschlossene Background-Notifications werden zu einem einzelnen Zähler gefaltet.
- **Syntax-Highlighting** — Upgrade auf highlight.js 11 für bessere Genauigkeit in Code Blocks, Diffs und File Previews.
- **Keyboard-Hints** — SSH von Mac → zeigt jetzt `opt`/`cmd` statt `alt`/`super`.
- **Verbesserter API-Retry-UX** — Ab dem zweiten Retry wird der Fehlergrund angezeigt; bei Überlastung erscheint ein Link zur Statusseite.
- **`/login` aus Agent-View** — Öffnet jetzt den Login-Dialog direkt aus der `claude agents`-Ansicht.
- **Subagent-Trust-Modell** — Subagents behandeln Nachrichten vom startenden Agent als normale Aufgaben; eine Agent-Nachricht gilt nie als Benutzer-Genehmigung.

#### Meta Engineering

**[Meta's AI Storage Blueprint at Scale](https://engineering.fb.com/2026/07/01/data-infrastructure/metas-ai-storage-blueprint-at-scale/)**

Meta beschreibt die vollständige Neuarchitektur ihres BLOB-Storage-Systems für KI-Workloads:

**Problem:** Legacy-Architektur hatte mehrere zustandsbehaftete Metadaten-Layer mit Latenzen von hunderten Millisekunden — für KI-Training unakzeptabel (GPU-Stalls).

**Lösung:**
- **Unified Metadata Schema:** Alle Metadaten in einem flachen Schema in ZippyDB → O(1)-Lookup statt verschachtelter Layer.
- **Kein Dataplane-Proxy:** Fat-Client-SDK streamt Daten direkt von Storage-Servern → besserer Durchsatz, weniger Energie.
- **Regionale Deployment:** BLOB-Storage jetzt als leichtgewichtiger, regionaler Service co-located mit GPUs.
- **Distributed Data Cache:** GPU-Host-Speicher als verteilter Cache → 80% Cache-Hit-Rate; Readplan-Metadaten in memcache → 1-2ms Metadaten-Zugriff.
- **Tiered Cache für Research-Velocity:** Daten werden on-demand von globalem HDD-Storage (L3: regionaler Flash) über L2 (GPU-Host-Flash) bis L1 (GPU-RAM) hydratisiert — ähnlich OS-Paging. Deep-Prefetch-API ermöglicht asynchrones Vorladen. Ergebnis: Ingestion-Zeit von Stunden auf Minuten reduziert.

**TechCrunch:** Meta plant außerdem einen eigenen Cloud-Infrastruktur-Dienst für AI-Compute, der mit AWS, Google Cloud und Azure konkurrieren soll.

### 4. 📄 ArXiv Papers

**[Measuring the Gap Between Human and LLM Research Ideas](http://arxiv.org/abs/2607.01233v1)**
Systematische Evaluation: Für 100 hochwertige Paper werden die wahrscheinlichen Inspirationsquellen reverse-engineered. LLMs werden dann gebeten, die Kerninnovation ausgehend von denselben Vorarbeiten zu generieren. Ergebnis: Quantifizierter Gap zwischen menschlicher und LLM-Ideenfindung in der Forschung.

**[Language-Critique Imitation Learning from Suboptimal Demonstrations](http://arxiv.org/abs/2607.01225v1)**
Ersetzt skalare Supervisionssignale (Konfidenz-Scores, Discriminator-Gewichte) durch natürlichsprachliche Kritiken. Das Framework beschreibt explizit Zwischenschritte, Fehlerarten und Korrekturmaßnahmen — strukturell reicher als bisherige Ansätze.

**[AutoMem: Automated Learning of Memory as a Cognitive Skill](http://arxiv.org/abs/2607.01224v1)**
LLMs lernen, ihr eigenes Gedächtnis zu managen: Dateisystem-Operationen werden als First-Class-Aktionen neben Task-Aktionen eingeführt. Das Modell entscheidet selbst, was es wann speichert und abruft — angelehnt an das kognitionswissenschaftliche Konzept der Metamemory.

**[Theoria: Rewrite-Acceptability Verification over Informal Reasoning States](http://arxiv.org/abs/2607.01223v1)**
Verifikations-Architektur zwischen Proof Assistants (sicher, aber begrenzt) und LLM-Judges (breit, aber opak): Eine Lösung wird in typisierte State-Transitions umgeschrieben, jede mit expliziter Begründung versehen und dann verifiziert.

**[The State-Prediction Separation Hypothesis](http://arxiv.org/abs/2607.01218v1)**
