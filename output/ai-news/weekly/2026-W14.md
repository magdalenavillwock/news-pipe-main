Fail-closed Remote Settings, interaktive Cloud-Setup-Wizards und granulare Kosten-Breakdowns zeigen: AI-Tools reifen vom Developer-Toy zum Enterprise-Produkt.

## 6. Anthropic & Claude — Detailbericht

### Claude Code v2.1.92 — Alle Features & Fixes

**Neue Features:**
- **`forceRemoteSettingsRefresh` Policy** — CLI blockiert Startup bis Remote-Settings geladen sind; Exit bei Fehler (fail-closed). Wichtig für Enterprise-Compliance.
- **Interaktiver Bedrock Setup Wizard** — Vom Login-Screen zugänglich ("3rd-party platform"). Führt durch AWS-Auth, Region-Config, Credential-Verification und Model-Pinning.
- **`/cost` Enhancement** — Per-Model und Cache-Hit Breakdown für Subscription-User. Erstmals Transparenz über tatsächliche Kosten pro Modell.
- **`/release-notes` Interactive Picker** — Interaktive Versions-Auswahl statt statischer Anzeige.
- **Remote Control Session-Namen** — Hostname als Default-Präfix (z.B. `myhost-graceful-unicorn`), überschreibbar mit `--remote-control-session-name-prefix`.
- **Prompt Cache Expiry Hinweis** — Footer-Hinweis wenn Cache nach Session-Rückkehr abgelaufen ist, zeigt ungefähre Token-Anzahl für nächsten uncached Turn.

**Bugfixes (12 Stück):**
- Subagent-Spawning nach tmux window kills/renumbering
- Prompt-type Stop hooks bei fast model `ok:false`
- Tool input validation bei JSON-encoded array/object fields
- Extended thinking whitespace-only blocks neben real content
- Feedback survey auto-submissions durch Keypresses
- Misleading "esc to interrupt" hints in fullscreen
- Homebrew update prompts für release channels
- `ctrl+e` jumping in multiline prompts
- Duplicate message positioning in fullscreen scroll (iTerm2, Ghostty)
- Plugin MCP servers "connecting" issues
- Write tool diff computation speed (+60% für files mit tabs/`&`/`$`)
- Removed: `/tag` und `/vim` commands (vim mode jetzt via `/config` → Editor mode)
